# ماڈیول 1: مشقیں اور کوئز

یہ سیکشن ماڈیول 1: روبوٹک نروس سسٹم (ROS 2) میں شاملہ تصورات کو سمجھنے کی جانچ کے لیے انٹرایکٹو مشقیں اور کوئز فراہم کرتا ہے۔

## مشقیں

### مشق 1.1: ROS 2 پبلشر-سبسکرائبر

**هدف**: `rclpy` کا استعمال کرتے ہوئے ایک سادہ ROS 2 پبلشر اور سبسکرائبر تیار کریں۔

1.  **ایک نیا ROS 2 پیکیج بنائیں**: اپنے ROS 2 ورک سپیس میں، `my_ros2_pkg` نام کا ایک نیا پائی تھون پیکیج بنائیں۔
    ```bash
    ros2 pkg create --build-type ament_python my_ros2_pkg
    ```
2.  **پبلشر نوڈ**: ایک پائی تھون اسکرپٹ (`my_ros2_pkg/my_ros2_pkg/publisher_node.py`) لکھیں جو `/chatter` نام کے ٹاپک پر `std_msgs.msg.String` میسجز 1 Hz کی شرح سے پبلش کرتا ہے۔ میسج ڈیٹا "Hello from Publisher: <count />" ہونا چاہیے۔
3.  **سبسکرائبر نوڈ**: ایک اور پائی تھون اسکرپٹ (`my_ros2_pkg/my_ros2_pkg/subscriber_node.py`) لکھیں جو `/chatter` ٹاپک کو سبسکرائب کرتا ہے اور وصول کردہ میسجز کو کنسول پر چھاپتا ہے۔
4.  **`setup.py` کو اپ ڈیٹ کریں**: یقینی بنائیں کہ آپ کی `setup.py` فائل دونوں نوڈس کے لیے اینٹری پوائنٹس کو درست طریقے سے بیان کرتی ہے۔
5.  **تعمیر اور چلائیں**: اپنے ROS 2 ورک سپیس کو تعمیر کریں اور دونوں نوڈس چلائیں۔ تصدیق کریں کہ سبسکرائبر نوڈ پبلشر سے میسجز وصول کر رہا ہے اور چھاپ رہا ہے۔
    ```bash
    cd <ros2_workspace>
    colcon build
    source install/setup.bash
    ros2 run my_ros2_pkg publisher_node
    ros2 run my_ros2_pkg subscriber_node
    ```

### مشق 1.2: ROS 2 سروس کلائنٹ-سرور

**هدف**: `rclpy` کا استعمال کرتے ہوئے ایک سادہ ROS 2 سروس کلائنٹ اور سرور نافذ کریں۔

1.  **سروس کی وضاحت**: ایک حسب ضرورت ROS 2 سروس میسج کی وضاحت کریں (مثلاً `my_ros2_pkg/srv/` میں `AddTwoInts.srv`) جو دو `int64` دلائل (`a`, `b`) لیتا ہے اور ایک `int64` کا مجموعہ (`result`) لوٹاتا ہے۔
2.  **سروس سرور**: ایک پائی تھون اسکرپٹ (`my_ros2_pkg/my_ros2_pkg/add_server.py`) لکھیں جو اس سروس کو نافذ کرتا ہے۔ یہ دو عدد وصول کرنا چاہیے، ان کا مجموعہ کرنا چاہیے، اور نتیجہ لوٹانا چاہیے۔
3.  **سروس کلائنٹ**: ایک اور پائی تھون اسکرپٹ (`my_ros2_pkg/my_ros2_pkg/add_client.py`) لکھیں جو ایک کلائنٹ کے طور پر کام کرتا ہے۔ یہ `add_two_ints` سروس کو دو اعداد کے ساتھ کال کرنا چاہیے اور لوٹائے گئے مجموعہ کو چھاپنا چاہیے۔
4.  **`setup.py` اور `package.xml` کو اپ ڈیٹ کریں**: ان فائلز کو حسب ضرورت سروس کی وضاحت کو شامل کرنے کے لیے تبدیل کریں۔
5.  **تعمیر اور چلائیں**: اپنے ROS 2 ورک سپیس کو تعمیر کریں اور سروس سرور چلائیں۔ پھر، سروس کو کال کرنے کے لیے کلائنٹ چلائیں۔ تصدیق کریں کہ کلائنٹ صحیح مجموعہ وصول کر رہا ہے۔
    ```bash
    cd <ros2_workspace>
    colcon build
    source install/setup.bash
    ros2 run my_ros2_pkg add_server
    # ایک نئی ٹرمنل میں
    ros2 run my_ros2_pkg add_client
    ```

## کوئز

### کوئز 1.1: ROS 2 تصورات

1.  ROS 2 میں غیر ہم وقت، کئی سے کئی ڈیٹا ایکسچینج کے لیے بنیادی مواصلت کا ذریعہ کیا ہے؟
    a) سروسز
    b) ایکشنز
    c) ٹاپکس
    d) پیرامیٹرز

2.  `rclpy` طریقہ جو پبلشر تیار کرنے کے لیے استعمال ہوتا ہے؟
    a) `create_subscriber()`
    b) `create_publisher()`
    c) `create_client()`
    d) `create_service()`

3.  URDF کا مطلب کیا ہے؟
    a) یونیورسل روبوٹ ڈیٹا فارمیٹ
    b) یونیفائیڈ روبوٹک ڈسکرپشن فائل
    c) یونیفائیڈ روبوٹ ڈسکرپشن فارمیٹ
    d) یونیورسل روبوٹکس ڈیفینیشن فریم ورک

4.  ROS 2 کمانڈ جو کسی ٹاپک پر پبلش کیے جانے والے میسجز کی تحقیق کرنے کے لیے استعمال ہوتی ہے؟
    a) `ros2 topic list`
    b) `ros2 run`
    c) `ros2 topic echo`
    d) `ros2 interface show`

### کوئز 1.2: `rclpy` اور پائی تھون ایجنٹس

1.  ROS 2 میں AI ایجنٹ کی یک جہتی کے لیے `rclpy` کا استعمال کرنے کا بنیادی فائدہ کیا ہے؟
    a) C++ کے مقابلے میں بہتر کارکردگی
    b) پائی تھون کے متنوع AI/ML ماحول تک رسائی
    c) کم سطحی ہارڈ ویئر کنٹرول کے لیے سادہ نحو
    d) ہم وقت کارروائیوں کے لیے اصلی حمایت

2.  ROS 2 پیکیج جو عام طور پر URDF اور جوائنٹ اسٹیٹس کی بنیاد پر روبوٹ کی کنیمیٹک حالت کو براڈ کاسٹ کرنے کے لیے استعمال ہوتا ہے؟
    a) `tf2_ros`
    b) `joint_state_publisher`
    c) `robot_state_publisher`
    d) `urdf_parser_py`
