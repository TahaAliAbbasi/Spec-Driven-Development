# باب 2: LLMs کے ساتھ کوگنیٹو منصوبہ بندی

یہ باب روبوٹکس میں کوگنیٹو منصوبہ بندی کے لیے بڑے زبان کے ماڈلز (LLMs) کے استعمال کو تلاش کرتا ہے، اس پر مرکوز ہے کہ LLMs قدرتی زبان کے کمانڈز کو انجام دینے والے روبوٹکس ایکشنز میں تبدیل کیسے کر سکتے ہیں۔ طلباء AI سسٹم نافذ کرنا سیکھیں گے جو کاموں کے بارے میں سوچ سکیں اور ایکشن سیکوئنس تیار کر سکیں۔ تکمیل کے بعد، طلباء کے قابل ہو جائیں گے:

*   **LLM-Based کوگنیٹو منصوبہ بندی کو سمجھیں**: وضاحت کریں کہ بڑے زبان کے ماڈلز روبوٹکس میں بلند سطح کے کام کی منصوبہ بندی اور استدلال کے لیے کیسے استعمال کیے جا سکتے ہیں۔
*   **پرامپٹ انجینئرنگ کی حکمت عملیاں ڈیزائن کریں**: مؤثر پرامپٹس تخلیق کریں جو LLMs کو قدرتی زبان کے کمانڈز سے مناسب روبوٹکس ایکشن سیکوئنس تیار کرنے کے لیے رہنمائی فراہم کرتے ہیں۔
*   **قدرتی زبان کو ROS 2 ایکشنز میں میپ کریں**: سسٹم نافذ کریں جو بلند سطح کے قدرتی زبان کے ہدایات کو مخصوص ROS 2 کمانڈز اور ایکشن سیکوئنس میں تبدیل کرتے ہیں۔
*   **کام کی تقسیم نافذ کریں**: روبوٹکس سسٹم کے لیے پیچیدہ کاموں کو سادہ، انجام دینے والے مراحل میں توڑنے کے لیے LLMs کا استعمال کریں۔
*   **LLMs میں سیاق و سباق اور میموری کو سنبھالیں**: کئی مراحل والے کاموں اور طویل المدت روبوٹکس آپریشنز کے لیے سیاق و سباق کی معلومات اور میموری کا نظم کریں۔
*   **روبوٹ کی صلاحیات کے ساتھ LLMs کو یکجا کریں**: یقینی بنائیں کہ LLM تیار کردہ منصوبے روبوٹ کی جسمانی اور عملی صلاحیات کو مدنظر رکھتے ہوئے قابلِ عمل ہیں۔
*   **خرابی کا انتظام اور بازیابی نافذ کریں**: سسٹم ڈیزائن کریں جو LLM تیار کردہ منصوبے ناکام ہونے پر پتہ لگا سکیں اور مناسب طریقے سے بازیاب ہو سکیں۔
*   **روبوٹکس کے لیے LLM جوابات کو بہتر بنائیں**: LLMs کو ساختہ، قابلِ عمل آؤٹ پٹس فراہم کرنے کے لیے تشکیل دیں جو روبوٹکس سسٹم کے لیے مناسب ہوں۔
*   **LLM-Based منصوبہ بندی کی معیار کا جائزہ لیں**: روبوٹکس ایپلی کیشنز میں LLM تیار کردہ منصوبوں کی مؤثرتا اور قابلِ اعتمادی کا جائزہ لیں۔

یہ سیکھنے کے نتائج اس باب کے مواد اور عملی مشق کی ہدایت کریں گے۔

## روبوٹکس کے لیے LLMs کے ساتھ کوگنیٹو منصوبہ بندی کا تعارف

بڑے زبان کے ماڈلز (LLMs) جیسے GPT-4، کلاؤڈ، یا اوپن-سورس متبادل روبوٹکس سسٹم کے لیے طاقتور کوگنیٹو انجن کے طور پر کام کر سکتے ہیں، بلند سطح کے کام کی منصوبہ بندی اور استدلال کو فعال کرتے ہیں۔ جب روبوٹکس میں لاگو کیا جاتا ہے، LLMs قدرتی زبان کے کمانڈز کی تشریح کر سکتے ہیں اور انہیں روبوٹکس سسٹم کے لیے انجام دینے والے ایکشنز کی ترتیب میں تبدیل کر سکتے ہیں۔ یہ صلاحیت انسان-روبوٹ تعامل کو مزید جدید بنانے کی طرف ایک اہم قدم ہے۔

### LLM-Based کوگنیٹو منصوبہ بندی کو سمجھنا

LLM-based کوگنیٹو منصوبہ بندی بڑے زبان کے ماڈلز کی استدلال کی صلاحیتوں کو استعمال کرتے ہوئے بلند سطح کے کاموں کو انجام دینے والے روبوٹکس ایکشنز میں توڑنے میں شامل ہے۔ روایتی قانون-م-based سسٹمز کے برعکس، LLMs ابہام کو سنبھال سکتے ہیں، سیاق و سباق کو سمجھ سکتے ہیں، اور پیچیدہ کاموں کے لیے تخلیقی حل تیار کر سکتے ہیں۔

کوگنیٹو منصوبہ بندی کا عمل عام طور پر شامل ہے:

1. **کمانڈ کی تشریح**: قدرتی زبان کے ان پٹ سے صارف کے ارادے کو سمجھنا
2. **کام کی تقسیم**: پیچیدہ کاموں کو سادہ، انجام دینے والے مراحل میں توڑنا
3. **سیاق و سباق کا ادراک**: ماحول اور روبوٹ کی موجودہ حالت کو مدنظر رکھنا
4. **ایکشن کی ترتیب**: مطلوبہ ہدف کو حاصل کرنے والے ایکشنز کی ایک ترتیب تیار کرنا
5. **پابندیوں کا انتظام**: روبوٹ کی جسمانی اور عملی پابندیوں کا احترام کرنا

### مؤثر پرامپٹ انجینئرنگ کی حکمت عملیاں ڈیزائن کرنا

LLM-based منصوبہ بندی کی مؤثرتا پر کام کے انداز کے پرامپٹس کا زیادہ انحصار ہوتا ہے۔ روبوٹکس ایپلی کیشنز کے لیے، پرامپٹس میں شامل ہونا چاہیے:

1. **کردار کی وضاحت**: LLM کا کردار صاف طور پر بیان کریں (مثلاً، "آپ ہیومنوائڈ روبوٹ کے لیے کوگنیٹو منصوبہ ساز ہیں")
2. **سیاق و سباق کی معلومات**: روبوٹ کی صلاحیات، موجودہ حالت، اور ماحول کے بارے میں معلومات فراہم کریں
3. **کام کی وضاحت**: کام کو واضح طور پر بیان کریں جو انجام دینا ہے
4. **آؤٹ پٹ فارمیٹ**: ایکشن سیکوئنس کے لیے متوقع فارمیٹ کی وضاحت کریں
5. **پابندیاں**: منصوبے کے لیے کوئی پابندیاں یا ضروریات کی وضاحت کریں

روبوٹکس منصوبہ بندی کے لیے مثالی پرامپٹ ساخت:

```
آپ ہیومنوائڈ روبوٹ کے لیے کوگنیٹو منصوبہ ساز ہیں۔ روبوٹ کے مندرجہ ذیل صلاحیات ہیں:
- نیویگیشن (مقامات پر جانا)
- اشیاء کا پتہ لگانا اور شناخت
- مینوپولیشن (اشیاء تھامنا، دروازے کھولنا/بند کرنا)
- رابطہ (بات کرنا، پیغامات ڈسپلے کرنا)

روبوٹ کی موجودہ حالت: روبوٹ لیونگ روم میں ہے، کچن کی طرف منہ کر کے۔ بیٹری کی سطح 85% ہے۔

کام: "کچن میں جائیں، کاؤنٹر پر لال سیب تلاش کریں، اور اسے مجھ تک لائیں۔"

براہ کرم JSON فارمیٹ میں ایکشنز کی ترتیب تیار کریں:
[
  {"action": "navigate_to", "target": "kitchen"},
  {"action": "detect_object", "object": "red apple", "location": "counter"},
  {"action": "grasp_object", "object": "red apple"},
  {"action": "navigate_to", "target": "user_location"},
  {"action": "release_object"}
]
```

### قدرتی زبان کو ROS 2 ایکشنز میں میپ کرنا

ROS 2 ماحول میں LLM-based منصوبہ بندی کو نافذ کرنے کے لیے، آپ کو ایک سسٹم تخلیق کرنا ہوگا جو LLM کے آؤٹ پٹ کو ROS 2 ایکشنز میں تبدیل کر سکے:

```python
import rclpy
from rclpy.node import Node
import json
import openai
from std_msgs.msg import String
from geometry_msgs.msg import Pose
from builtin_interfaces.msg import Duration

class CognitivePlannerNode(Node):
    def __init__(self):
        super().__init__('cognitive_planner_node')

        # بلند سطح کے کمانڈز کے لیے پبلشر
        self.action_publisher = self.create_publisher(String, 'robot_actions', 10)

        # قدرتی زبان کے کمانڈز کے لیے سبسکرائبر
        self.command_subscriber = self.create_subscription(
            String,
            'natural_language_command',
            self.command_callback,
            10
        )

        # OpenAI API کلید شروع کریں
        openai.api_key = "YOUR_API_KEY_HERE"

        self.get_logger().info('Cognitive Planner Node Started')

    def command_callback(self, msg):
        """قدرتی زبان کا کمانڈ پروسیس کریں اور ایکشن سیکوئنس تیار کریں"""
        natural_language_command = msg.data

        # LLM کا استعمال کرتے ہوئے منصوبہ تیار کریں
        action_sequence = self.generate_plan(natural_language_command)

        if action_sequence:
            # ایکشن سیکوئنس پبلش کریں
            action_msg = String()
            action_msg.data = json.dumps(action_sequence)
            self.action_publisher.publish(action_msg)

            self.get_logger().info(f'Published action sequence: {action_sequence}')

    def generate_plan(self, command):
        """LLM کا استعمال کرتے ہوئے ایکشنز کی ترتیب تیار کریں"""
        prompt = f"""
        آپ ہیومنوائڈ روبوٹ کے لیے کوگنیٹو منصوبہ ساز ہیں۔ روبوٹ مندرجہ ذیل ایکشنز انجام دے سکتا ہے:
        - navigate_to: مخصوص مقام پر جانا
        - detect_object: ماحول میں اشیاء کی شناخت
        - grasp_object: اشیاء تھامنا
        - release_object: اشیاء چھوڑنا
        - speak: لفظی رابطہ
        - open_door: دروازہ کھولنا
        - close_door: دروازہ بند کرنا

        روبوٹ کی موجودہ حالت: روبوٹ لیونگ روم میں ہے، کچن کی طرف منہ کر کے۔ بیٹری کی سطح 85% ہے۔

        کام: "{command}"

        براہ کرم JSON فارمیٹ میں ایکشنز کی ترتیب تیار کریں:
        [
            {{"action": "...", "target": "...", "object": "..."}}
        ]
        """

        try:
            response = openai.ChatCompletion.create(
                model="gpt-3.5-turbo",  # یا gpt-4 بہتر کارکردگی کے لیے
                messages=[{"role": "user", "content": prompt}],
                max_tokens=500,
                temperature=0.3
            )

            # جواب سے ایکشن سیکوئنس نکالیں
            response_text = response.choices[0].message['content'].strip()

            # جواب میں JSON کا حصہ تلاش کریں
            start_idx = response_text.find('[')
            end_idx = response_text.rfind(']') + 1

            if start_idx != -1 and end_idx != 0:
                json_str = response_text[start_idx:end_idx]
                action_sequence = json.loads(json_str)

                return action_sequence
            else:
                self.get_logger().error(f'LLM response سے JSON نکال نہیں سکے: {response_text}')
                return None

        except Exception as e:
            self.get_logger().error(f'منصوبہ تیار کرنے میں خرابی: {e}')
            return None

def main(args=None):
    rclpy.init(args=args)
    node = CognitivePlannerNode()

    try:
        rclpy.spin(node)
    except KeyboardInterrupt:
        pass
    finally:
        node.destroy_node()
        rclpy.shutdown()

if __name__ == '__main__':
    main()
```

### LLMs کے ساتھ کام کی تقسیم

LLMs پیچیدہ کاموں کو سادہ مراحل میں توڑنے میں ماہر ہیں۔ یہ روبوٹکس کے لیے اہم ہے جہاں پیچیدہ اہداف کو انجام دینے والے ایکشنز میں توڑنا ضروری ہے۔ مثال کے طور پر، "ڈنر کے لیے ٹیبل سیٹ کریں" جیسے کمانڈ کو مندرجہ ذیل میں توڑا جا سکتا ہے:

1. کچن میں جائیں
2. پلیٹس، کپس، اور اوزاروں کی شناخت کریں
3. پلیٹ تھامیں
4. ڈائننگ ٹیبل میں جائیں
5. ٹیبل پر پلیٹ رکھیں
6. دیگر اشیاء کے لیے دہرائیں

### سیاق و سباق اور میموری کا نظم

مؤثر منصوبہ بندی کے لیے، LLMs کو مندرجہ ذیل کے بارے میں سیاق و سباق برقرار رکھنے کی ضرورت ہے:

- روبوٹ کا موجودہ مقام اور حالت
- ماحول کی ترتیب
- پہلے دیکھی گئی اشیاء
- کام کی پیش رفت

اسے مندرجہ ذیل کے ذریعے حاصل کیا جا سکتا ہے:

1. **پرامپٹ انجینئرنگ**: ہر پرامپٹ میں متعلقہ سیاق و سباق شامل کرنا
2. **میموری سسٹم**: متعلقہ معلومات کو ذخیرہ کرنے اور بازیافت کرنے کے لیے بیرونی میموری نافذ کرنا
3. **حالت ٹریکنگ**: دنیا کی حالت کی ایک نمائندگی برقرار رکھنا جسے اپ ڈیٹ کیا جا سکے اور استفسار کیا جا سکے

### روبوٹ کی صلاحیات کے ساتھ یک جہتی

LLM تیار کردہ منصوبے کو روبوٹ کی جسمانی اور عملی صلاحیات کو مدنظر رکھتے ہوئے قابلِ عمل ہونا چاہیے۔ اس کے لیے ضروری ہے:

1. **صلاحیت کی ماڈلنگ**: یہ واضح طور پر بیان کریں کہ روبوٹ کیا کر سکتا ہے اور کیا نہیں کر سکتا
2. **قابلیت کی جانچ**: یہ تصدیق کریں کہ تجویز کردہ ایکشنز روبوٹ کی صلاحیات کے اندر ہیں
3. **واپسی کی منصوبہ بندی**: جب بنیادی منصوبے ناکام ہوں تو متبادل حکمت عملیاں رکھنا

### خرابی کا انتظام اور بازیابی

LLM-based منصوبہ بندی کے سسٹم کو مضبوط خرابی کا انتظام درکار ہے:

1. **ایکشن کی توثیق**: یہ تصدیق کریں کہ ایکشنز موجودہ سیاق و سباق میں معنی رکھتے ہیں
2. **ناکامی کا پتہ لگانا**: ایکشنز کے انجام نہ دینے کا پتہ لگانے کے لیے شناخت کریں
3. **منصوبے کی اصلاح**: غیر متوقع صورتحال پیش آنے پر منصوبوں کو مدون کریں
4. **انسانی مداخلت**: ضرورت پڑنے پر انسانی مدد کے لیے میکانزم فراہم کریں

ان تکنیکوں کو نافذ کر کے، آپ مہارت والے کوگنیٹو منصوبہ بندی کے سسٹم تخلیق کر سکتے ہیں جو روبوٹکس کام کی انجام دہی کے لیے LLMs کی طاقت کا فائدہ اٹھاتے ہیں۔

## عام LLM یک جہتی کے مسائل کو حل کرنا

کوگنیٹو منصوبہ بندی کے لیے روبوٹکس سسٹم کے ساتھ LLMs کو یکجا کرتے وقت، کئی عام مسائل پیش آ سکتے ہیں۔ یہاں کچھ حل ہیں:

### API کنکشن اور توثیق کے مسائل

- **غلط API کلید**: یقینی بنائیں کہ آپ کی LLM API کلید (OpenAI، Anthropic، وغیرہ) درست سیٹ ہے اور اس میں کافی کریڈٹس ہیں۔
- **نیٹ ورک کنکٹیویٹی**: یقینی بنائیں کہ روبوٹ کے پاس API کالز کے لیے مستحکم انٹرنیٹ کنکشن ہے، خاص طور پر حقیقی وقت کی منصوبہ بندی کے لیے اہم۔
- **شرح کی حد**: API کی حد کے مسائل سے نمٹنے کے لیے ایکسپونینشل بیک آف کے ساتھ دوبارہ کوشش کا منطق اور کوائف کی قطار نافذ کریں۔

### منصوبہ بندی کی معیار کے مسائل

- **غیر مسلسل آؤٹ پٹس**: زیادہ مسلسل منصوبہ بندی کے آؤٹ پٹس کے لیے کم درجہ (0.1-0.3) استعمال کریں۔
- **غیر متعلقہ ایکشنز**: روبوٹ کی صلاحیات اور ماحول کی پابندیوں کے بارے میں مزید سیاق و سباق فراہم کر کے پرامپٹ انجینئرنگ میں بہتری لائیں۔
- **غیر قابلِ عمل منصوبے**: یہ تصدیق کرنے کے لیے قابلیت کی جانچ نافذ کریں کہ منصوبہ بند کردہ ایکشنز روبوٹ کی صلاحیات کے اندر ہیں۔

### سیاق و سباق کا نظم کے مسائل

- **گم شدہ حالت کی معلومات**: روبوٹ کے مقام، اٹھائی گئی اشیاء، وغیرہ کا خیال رکھنے کے لیے مناسب حالت ٹریکنگ اور سیاق و سباق کا نظم نافذ کریں۔
- **سیاق و سباق کی ونڈو کی حدیں**: طویل المدت کاموں کے لیے، متعلقہ معلومات کو ذخیرہ کرنے اور بازیافت کرنے کے لیے بیرونی میموری سسٹم نافذ کریں۔
- **متعارض منصوبے**: متوازی یا متعارض کمانڈز کو مناسب طریقے سے سنبھالنے کے لیے سسٹم ڈیزائن کریں۔

### کارکردگی کی بہتری

- **دیر کے مسائل**: API کالز اور جواب کے اوقات کو کم کرنے کے لیے عام کمانڈز کے جوابات کو کیش کریں۔
- **لاگت کا نظم**: ٹوکن استعمال کو نظر رکھیں اور سادہ منصوبہ بندی کے کاموں کے لیے چھوٹے ماڈلز کا استعمال غور کریں۔
- **متوازی پروسیسنگ**: روبوٹ کے کاموں کو بلاک کرنے سے بچنے کے لیے منصوبہ بندی کی درخواستوں کو غیر ہم وقت میں پروسیس کریں۔

## پیداواری انتظام کے لیے بہترین طریقے

### سیکیورٹی کے اہمات

- ماحولیاتی متغیرات یا محفوظ والٹس کا استعمال کرتے ہوئے API کلیدز کو محفوظ طریقے سے ذخیرہ کریں۔
- خراب کمانڈز کو روکنے کے لیے LLM تیار کردہ کمانڈز کی مناسب توثیق نافذ کریں۔
- روبوٹ اور LLM سروسز کے درمیان ارسال کے دوران حساس ڈیٹا کو خفیہ کریں۔

### قابلِ اعتمادی اور ویپسیاں

- جب LLM سروسز غیر دستیاب ہوں تو ویپسی کی منصوبہ بندی کی حکمت عملیاں نافذ کریں۔
- AI منصوبہ بندی ناکام ہونے پر قانون-based یا پیش-پروگرام کردہ رویوں کی طرف نرم انحدام ڈیزائن کریں۔
- اہم حفاظتی کاموں کے لیے مقامی منصوبہ بندی کی صلاحیات برقرار رکھیں۔

### نگرانی اور لاگنگ

- ڈیبگنگ اور کارکردگی کے تجزیہ کے لیے تمام LLM تعاملات کو لاگ کریں۔
- منصوبہ بندی کی کامیابی کی شرح کو نظر رکھیں اور عام ناکامی کے نمونے شناخت کریں۔
- معاش (ٹوکن، دیر) کو کارکردگی کے اخراجات اور بہتری کے لیے ٹریک کریں۔
