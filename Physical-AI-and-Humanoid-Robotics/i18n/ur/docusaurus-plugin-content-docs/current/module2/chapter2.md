# باب 2: یونٹی کی یک جہتی - سیکھنے کے نتائج

یہ باب یونٹی کی یک جہتی کو تلاش کرتا ہے، ایک طاقتور حقیقی وقت 3D ترقی کا پلیٹ فارم، ROS 2 کے ساتھ زیادہ معیار کی روبوٹکس سیمولیشن، اعلیٰ وژولائزیشن، اور سمجھدار انسان-روبوٹ تعامل (HRI) کے لیے۔ تکمیل کے بعد، طلباء کے قابل ہو جائیں گے:

*   **روبوٹکس میں یونٹی کے کردار کو سمجھیں**: وضاحت کریں کہ یونٹی گیزبو اور دیگر سیمولیشن ٹولز کو کیسے مکمل کرتا ہے جو اعلیٰ رینڈرنگ، حقیقی ماحول، اور تعاملاتی HRI صلاحیتوں کی فراہمی کرتا ہے۔
*   **ROS 2 کی یک جہتی کے لیے یونٹی سیٹ اپ کریں**: یونٹی پروجیکٹ کو ROS 2 سسٹم کے ساتھ مواصلت کے لیے تشکیل دیں، `Unity-ROS-TCP-Connector` یا `ROS-Unity-Integration` جیسے پیکیجز کا استعمال کریں۔
*   **زیادہ معیار کی روبوٹ وژولائزیشنز تیار کریں**: یونٹی مناظر تیار کریں تاکہ روبوٹ ماڈلز کی نمائندگی حقیقی ٹیکسچرز، لائٹنگ، اور ماحولیاتی تفصیلات کے ساتھ کی جا سکے تاکہ بصری معیار میں اضافہ ہو سکے۔
*   **یونٹی میں ROS 2 مواصلت نافذ کریں**: یونٹی کے اندر C# اسکرپٹس لکھیں تاکہ ROS 2 پبلشرز اور سبسکرائبرز تخلیق کیے جا سکیں، یونٹی اور ROS 2 نوڈس کے مابین دو طرفہ ڈیٹا ایکسچینج کو فعال کر سکیں۔
*   **انسان-روبوٹ تعامل (HRI) انٹرفیسز ڈیزائن کریں**: یونٹی میں تعاملاتی صارف انٹرفیسز تیار کریں جو انسانوں کو سیمولیٹڈ روبوٹس کو کنٹرول کرنے یا ان کی حالت کو حقیقی وقت میں دیکھنے کی اجازت دیں۔
*   **اعلیٰ سینسر ڈیٹا کو سیمولیٹ کریں**: یونٹی کو ROS 2 ٹاپکس پر حقیقی سینسر ڈیٹا (مثلاً کیمرہ فیڈز، ڈیپتھ میپس، آبجیکٹ ڈیٹیکشن) تیار کرنے اور پبلش کرنے کے لیے تشکیل دیں۔
*   **کو-سیمولیشن کی حکمت عملیں تیار کریں**: سیکھیں کہ پیچیدہ کو-سیمولیشن منظرناموں کے لیے گیزبو کے فزکس انجن کے ساتھ یونٹی کی رینڈرنگ اور HRI صلاحیتوں کو کیسے جوڑا جائے۔

یہ سیکھنے کے نتائج اس باب کے مواد اور عملی مشق کی ہدایت کریں گے۔

## روبوٹکس کے لیے یونٹی کی یک جہتی کا تعارف

یونٹی، ایک نمایاں حقیقی وقت 3D ترقی کا پلیٹ فارم، بصری طور پر امیر اور تعاملاتی سیمولیشن تخلیق کرنے کے لیے بے مثال صلاحیتوں کی فراہمی کرتا ہے۔ جب روبوٹکس مڈل ویئر جیسے ROS 2 کے ساتھ یکجا کیا جاتا ہے، یونٹی زیادہ معیار کی وژولائزیشن، انسان-روبوٹ تعامل (HRI) کی ترقی، اور یہاں تک کہ اعلیٰ سینسر سیمولیشن کے لیے ایک طاقتور ٹول میں تبدیل ہو جاتا ہے۔

### ROS 2 کے ساتھ یونٹی کیوں یکجا کریں؟

*   **زیادہ معیار کی رینڈرنگ**: یونٹی کا اعلیٰ گرافکس انجن فوٹو ریلسٹک ماحول تخلیق کرنے کی اجازت دیتا ہے، جو روبوٹکس سیمولیشن کی بصری حقیقت پسندی کو گیزبو جیسے فزکس-م-centric سیمولیٹرز کے مقابلے میں بہت زیادہ بہتر بناتا ہے۔
*   **امیر صارف انٹرفیسز**: یونٹی سمجھدار اور تعاملاتی صارف انٹرفیسز تیار کرنے میں ماہر ہے۔ یہ اعلیٰ درجے کے انسان-روبوٹ تعامل (HRI) ایپلی کیشنز تیار کرنے کے لیے اہم ہے، جو آپریٹرز کو روبوٹس کو زیادہ آسانی سے مانیٹر اور کنٹرول کرنے کی اجازت دیتا ہے۔
*   **کراس-پلیٹ فارم ڈیپلائمنٹ**: یونٹی ڈیسک ٹاپ اور موبائل سے لے کر VR/AR تک کے وسیع پیمانے پر پلیٹ فارمزم کے لیے ڈیپلائمنٹ کی حمایت کرتا ہے، جو مختلف روبوٹکس ایپلی کیشنز اور تربیت کے منظرناموں کے لیے اسے لچکدار بنا دیتا ہے۔
*   **اعلیٰ سینسر سیمولیشن**: جبکہ گیزبو مضبوط فزکس-م-based سینسر ماڈلز فراہم کرتا ہے، یونٹی زیادہ معیار کا سینسر ڈیٹا تیار کر سکتا ہے، خاص طور پر کیمرے، ڈیپتھ سینسرز، اور آبجیکٹ ڈیٹیکشن کے لیے، اس کی رینڈرنگ کی صلاحیتوں کا استعمال کرتے ہوئے۔
*   **تیز پروٹو ٹائپنگ**: یونٹی کا کمپونینٹ-م-based آرکیٹیکچر اور وسیع اثاثہ اسٹور روبوٹ ڈیزائن، ماحول، اور کنٹرول انٹرفیسز کی تیز پروٹو ٹائپنگ کو فعال کرتا ہے۔
*   **کو-سیمولیشن کی صلاحیتیں**: یونٹی کو دیگر سیمولیٹرز (مثلاً گیزبو) کے ساتھ کو-سیمولیشن سیٹ اپ میں استعمال کیا جا سکتا ہے، جہاں ہر سیمولیٹر وہ پہلو سنبھالتا ہے جس میں وہ ماہر ہے (مثلاً گیزبو فزکس کے لیے، یونٹی رینڈرنگ کے لیے)۔

### ROS 2 کے لیے یونٹی کا سیٹ اپ

یونٹی کو ROS 2 کے ساتھ یکجا کرنا عام طور پر ان خصوصی پیکیجز کو شامل کرتا ہے جو دو سسٹم کے مابین ڈیٹا ایکسچینج کو آسان بناتے ہیں۔ `Unity-ROS-TCP-Connector` (ROS 1 اور ROS 2 کے لیے) یا `ROS-Unity-Integration` (ROS 2 کے لیے مخصوص، یونٹی روبوٹکس ہب کا حصہ) عام انتخابات ہیں۔

**سیٹ اپ کے لیے کلیدی اقدامات**:

1.  **یونٹی ہب اور یونٹی ایڈیٹر انسٹال کریں**: یقینی بنائیں کہ آپ کے پاس یونٹی کا مطابقت رکھنے والا ورژن انسٹال ہے۔
2.  **ایک نیا یونٹی پروجیکٹ بنائیں**: یونٹی میں ایک نیا 3D پروجیکٹ شروع کریں۔
3.  **ROS 2 یک جہتی پیکیج درآمد کریں**: اپنے یونٹی پروجیکٹ میں `Unity-ROS-TCP-Connector` یا `ROS-Unity-Integration` پیکیج شامل کریں۔ ان پیکیجز میں عام طور پر ROS 2 میسج سیریلائزیشن/ڈی سیریلائزیشن اور TCP مواصلت کے لیے C# اسکرپٹس فراہم کیے جاتے ہیں۔
4.  **`ROSConnection` تشکیل دیں**: یونٹی گیم آبجیکٹ میں `ROSConnection` کمپونینٹ سیٹ اپ کریں، ROS 2 ماسٹر URI (یا TCP کنیکشن کے لیے IP ایڈریس اور پورٹ) کی وضاحت کریں۔
5.  **حسب ضرورت ROS 2 میسج ہینڈلرز تخلیق کریں**: حسب ضرورت ROS 2 میسجز کے لیے، آپ کو ان کی C# مساوی وضاحت کرنے کی ضرورت ہوگی یونٹی میں اور سیریلائزیشن/ڈی سیریلائزیشن منطق لکھنے کی ضرورت ہوگی۔

#### مثال: یونٹی میں بنیادی ROS 2 پبلشر ( تصوراتی C#)

یونٹی سے ROS 2 میں ڈیٹا بھیجنے کے لیے، آپ ایک پبلشر اسکرپٹ تخلیق کریں گے:

```csharp
using UnityEngine;
using RosMessageTypes.Std;
using Unity.Robotics.ROSTCPConnector;

public class UnityPublisher : MonoBehaviour
{
    ROSConnection ros;
    public string topicName = "/unity_chatter";
    private float publishMessageFrequency = 1.0f;
    private float timeElapsed;

    void Start()
    {
        ros = ROSConnection.GetOrCreateInstance();
        ros.RegisterPublisher<StringMsg>(topicName);
    }

    void Update()
    {
        timeElapsed += Time.deltaTime;
        if (timeElapsed > publishMessageFrequency)
        {
            StringMsg chatter = new StringMsg("Hello from Unity!");
            ros.Publish(topicName, chatter);
            timeElapsed = 0;
        }
    }
}
```

#### مثال: یونٹی میں بنیادی ROS 2 سبسکرائبر ( تصوراتی C#)

یونٹی میں ROS 2 سے ڈیٹا وصول کرنے کے لیے، آپ ایک سبسکرائبر اسکرپٹ تخلیق کریں گے:

```csharp
using UnityEngine;
using RosMessageTypes.Geometry;
using Unity.Robotics.ROSTCPConnector;

public class UnitySubscriber : MonoBehaviour
{
    public string topicName = "/ros_vel_cmd";

    void Start()
    {
        ROSConnection.GetOrCreateInstance().Subscribe<TwistMsg>(topicName, CmdVelReceived);
    }

    void CmdVelReceived(TwistMsg twistMessage)
    {
        Debug.Log($"Received ROS 2 Twist message: Linear X={twistMessage.linear.x}, Angular Z={twistMessage.angular.z}");
        // یونٹی میں سیمولیٹڈ روبوٹ پر ویلوسٹی لاگو کریں
    }
}
```

### زیادہ معیار کی رینڈرنگ اور انسان-روبوٹ تعامل (HRI)

یونٹی کی رینڈرنگ کی صلاحیتیں سمجھدار روبوٹ وژولائزیشنز اور HRI تجربات تخلیق کرنے کے لیے بے مثال ہیں۔ اس میں شامل ہے:

*   **حقیقی ماحول**: تفصیلی مجازی دنیا تیار کرنا جو حقیقی دنیا کے منظرناموں سے قریب سے مماثل ہو، جو محسوس کن تربیت اور دور دراز کے آپریشن کے لیے اہم ہے۔
*   **حسب ضرورت روبوٹ ماڈلز**: حقیقی ٹیکسچرز، مواد، اور جوائنٹ حرکات کے ساتھ پیچیدہ روبوٹ ماڈلز کو درآمد اور اینیمیٹ کرنا۔
*   **تعاملاتی کنٹرولز**: سلائیڈرز، بٹن، اور جوائسٹکس کے ساتھ گرافیکل صارف انٹرفیسز (GUIs) تیار کرنا جو ROS 2 کو کمانڈز بھیجتے ہیں، سمجھدار روبوٹ ٹیلی آپریشن کی اجازت دیتے ہیں۔
*   **ڈیٹا وژولائزیشن**: حقیقی وقت کے سینسر ڈیٹا (مثلاً پوائنٹ کلاؤڈز، کیمرہ فیڈز، روبوٹ کی حالت) کو براہ راست یونٹی منظر یا اوور لے ڈیش بورڈز پر ظاہر کرنا۔

یونٹی کی مضبوطیوں کا فائدہ اٹھاتے ہوئے، ڈیولپرز جسمانی مصنوعی ذہانت اور ہیومنوائڈ روبوٹکس کے لیے امیر، دلچسپ، اور انتہائی قابل عمل ڈیجیٹل ٹوئن ایپلی کیشنز تخلیق کر سکتے ہیں۔
